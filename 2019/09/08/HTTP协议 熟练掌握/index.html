HTTP 协议熟练掌握

HTTP基础概念

  • HTTP 是什么？ 不是什么？
    超文本传输协议. http是互联网在两点之间传输文本、图片、音频、视频等超文本的规范和约定。
    http不是软件、也不是互联网、也不是编程语言

  • TCP/IP
    1)TCP/IP 协议 四层(从上到下)：应用层、传输层、网际层、链接层

    2)IP协议：网际层。只要用来解决寻址、路由问题，以及如何在两点之间传输数据包。
    ipv4版，总共有2^32 大约42亿个可分配的地址。 ipv6版有 2^128个

    3)TCP协议：传输层。基于ip协议的 可靠的(数据不丢失)、字节流(数据完整的) 通信。
    HTTP是一个“传输协议”。不用去关心寻址、路由、数据完整性等传输细节。HTTP是运行在TCP/IP上。更准确的来说是HTTP over TCP/IP.

  • DNS
    1)使用TCP/IP协议还通信是得使用ip地址，所以需要把域名做一个转换
    2)DNS是为了方便记忆.

  • URL/URI
    1)DNS和IP只是标记了互联网的主机.但是却没办法准确定位到相应的资源(如文本、图片、页面)
    2)URL组成三部分： 协议名+主机+路径
    3)URI(uniform resource identifier) 统一资源标识符。能够唯一标识互联网资源.URL(uniform resource locator) 统一资源定位符，也就是我们常说的网址。几乎相同不用严格区分。

  • HTTPS
    1)运行在SSL/TLS之上的HTTP
    2)SSL/TLS 是建立在TCP/IP之上负责的负责加密通信的安全协议,是可靠的传输协议，也用作HTTP的下层。

  • TCP/IP 四层网络分层模型/ OSI 七层网络分层模型

Level Layer Function
L4 应用层 面向应用传输数据
L3 传输层 TCP/UDP就在此层。保证数据在IP标记的两点之间的可靠传输。
L2 网际层 ip协议就在此层。在链接层的基础上, 用ip地址来取代MAC地址。把许多广域网/局域网形成一个虚拟巨大的网络，在这个网络找设备只需要把ip地址“翻译”成MAC地址就可以了。
L1 链接层 接收以太网、wifi上传的原始数据包。工作在网卡层次,用MAC来标记网络上的设备。也叫MAC层。

remarks: UDP和TCP区别 --> TCP是有状态的，需要先与对方建立连接才能传输数据，保证数据不重复不丢失，传输的数据是连续的字节流数据，有序。 UDP是无状态的，无需先建立链接就可以传输数据，但不保证数据一定能发送到对方。传输的数据是分散的数据包，有序发，乱序收。

Level Layer Function
L7 应用层 面向应用传输数据
L6 表现层 把会话转换成可理解的语法语义
L5 会话层 维护网络中的连接状态，即保持会话和同步
L4 传输层 同TCP/IP四层中的传输层
L3 网际层 同TCP/IP四层中的网际层
L2 链接层 同TCP/IP四层中的链接层
L1 物理层 网络的物理形式。如电缆、光纤、网卡、集线器。

remarks: MAC层传输的单位是帧(frame)，IP层传世的单位是包(packet)，TCP层传输的单位是段(segment)， HTTP层传输的单位消息或者报文(message). 这些没有本质的区别，都可以统称为数据包

  • 四层负载均衡/七层负载均衡
    1)四层负载均衡：工作在传输层,基于TCP/IP协议。 例如 IP地址、端口号等对后端服务器进行的负载均衡。
    2)七层负载均衡：工作在应用层，看到的是HTTP协议。解析HTTP协议里面的URL、主机名资源等数据，再以合适的策略来实现负载均衡。
    remarks: 绝大部分 由操作系统处理的是四层或者四层一下。需要程序自己处理的绝大部分是七层

  • 二层转发、三层路由
    这里的二层三层都指的是OSI七层模型中的。 二层转发：指工作在数据连接层的设备(如交换机)通过找到设备的MAC地址进行数据转发。
    三层路由指的是工作在网络层的设备(如路由器)，通过解析数据包头信息，找到ip地址，从而进行转发数据。
