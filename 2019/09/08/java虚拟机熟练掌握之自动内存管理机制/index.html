Java虚拟机熟练掌握

JVM运行时数据区域(包含jdk1.7后的改动)已及内存溢出溢出的解决办法

  • 程序计数器
    1)程序计数器是线程私有
    2)虚拟机的概念模型中，字节码解释器就是通过改变程序计数器的值来取下一条需要执行的字节码指令
    3)很小的内存，可以看做当前线程所执行的字节码行号指示器
    4)程序计数器是java虚拟机规范中唯一没有规定任务OutOfMemoryError区域。(其他区域都有可能出现OOM,后续会详细写一下其他区域出现OOM的情况)

  • 虚拟机栈
    1)虚拟机栈是线程私有
    2)虚拟机栈描述的是java方法执行的内存模型：在方法执行的时候虚拟机会创建一个栈帧：用于存储局部变量表、操作数栈、动态链接、方法出口等信息。一个方法的执行到结束也就对应着栈帧的入栈和出栈。
    remarks: 局部变量表 用于存放 基础类型、对象引用、returnAddress类型。且局部变量表所需内存空间在编译期间就完成分配

  • 本地方法栈
    1)本地方法栈和虚拟机栈功能和作用类似，区别在于虚拟机栈执行的是java方法, 而本地方法栈执行的是Native方法。
    remarks: Hotspot虚拟机将本地方法栈和虚拟机栈合二为一

  • Java堆
    1)java堆是线程共享的
    2)堆区是用来存放对象实例，几乎所有的对象实例都在堆区分配内存.(所有的对象实例和数组都要在堆上分配)
    3)-XMN 设置java新生代大小、-XMX 设置java最大堆内存、-XMS JVM启动时分配的内存、-Xss JVM启动为每个线程分配的内存大小

  • 方法区
    1)方法区是线程共享的
    2)用于存储已被虚拟机加载的类型信息、常量、即时编译器编译后的代码缓存等数据
    3)jdk1.7之后对方法区做了几个比较大的变动:

  • 字符串常量池(interned strings)转移到了Java heap.

  • 类的静态变量(class statics variables)转移到了Java heap

  • 取消了永久代，方法存放于元数据空间。设置参数由PermSize MaxPermSize -> 元空间参数MetaspaceSize

  • 运行时常量池
    1)运行时常量池是方法区的一部分。
    2)用于存放编译器生成的各种字面量和符号引用。在类加载后进入方法区的运行时常量池中存放。
    3)不仅会保存class文件中描述的符号引用, 还会把翻译出来的直接引用也存储在运行在运行时常量池中

对象的创建、布局、访问

  • 普通java对象(不包括数组(数组的加载有些不同,后续会详细写)和class对象)创建过程:
    1)当虚拟机收到一个new的指令的时候，会先去常量池中检查是否能找到一个这个指令参数的类符号引用。
    找不到的话，就执行队友的类加载过程(加载->准备->验证->解析->初始化->使用->卸载).后面会详细写类加载过程。

  • 类加载完成后,虚拟机将会给对象分配内存
    1)内存分配的两种策略之1: 指针碰撞 (java内存是规整的)
    2)内存分配的两种策略之2: 空闲列表 (java内存不是规整的)
    remarks : Serial、ParNew等带有Compact(压缩)过程的垃圾收集器采用的是内存分配策略是: 指针碰撞。 而基于Mark-Sweep算法的如CMS就使用的策略是 空闲列表

  • 分配内存怎么保证线程安全
    场景：在内存分配的时候,线程A和线程B同时操作指针进行移动，这时候就会出现指针只移动了一次的线程不安全问题。
    1)对内存的分配动作进行同步处理: 虚拟机采用cas已及失败重试机制保证了更新操作(移动指针)的原子性。
    2)使用堆区里面的TLAB(Thrad Local Allocate Buffer). 即每个线程在堆中预先分配一块内存(TLAB),这样做的
    目的是将内存分配动作按照线程划分在不同的空间之中

  • 内存分配完成后续
    1)内存分配完成后，虚拟机将分配到的内存空间都初始化为零值，不包括对象头。(这样做是为了
    对象的实例字段在java代码中不被赋初值就可以使用。如我们自己定义的类大部分就不用赋初值)
    2)然后虚拟机会对对象头进行初始化。如 这个对象时哪个类的实例、如何才能找到类的元数据信息
    、对象的哈希码、GC年龄等信息都存放在对象的对象头中。
    3)执行new指令后,后续会紧跟着 init 方法 （构造函数在 init 后执行)
    remarks: <init> 方法是由编译器自动收集所有类变量(static修饰的类字段)的赋值动作和静态语句块

  • 对象的内存布局
    对象在内存的布局分为3个部分: 1.对象头 2.实例数据 3.对齐填充
    1)对象头: 对象头存放两部分数据： 1.运行时候数据。 2 类型指针

  • 运行时数据包括 hashCode、GC年龄代、锁状态标志、线程持有锁、偏向线程ID、偏向时间戳. 官网称之为“Mark Word”。在32位和64位虚拟机(未开启指针压缩)中的内存为 32bit、64bit(开启指针压缩后也变成了32bit)

  • 类型指针, 即对象指向它元数据的指针, 作用是用来找到这个对象是哪个类的实例。(值得注意的一点是，并不是所有的虚拟机)
